# Agent Persona and Error UX Contract

## Purpose

Long John Silver's assistant persona is defined by the bundled package at
`config/personas/default/`. Future personas should use the same folder-package
shape: `persona.md`, `persona.json`, optional avatar, and optional `theme.json`.
User-facing assistant responses should feel like the selected competent
quartermaster regardless of where the text comes from: the LLM, a clarification
builder, a websocket fallback, or a deterministic plan failure.

## Prompt path

All normal user-facing LLM replies must be built through
`PromptBuilder.build_system_prompt()`. The prompt builder resolves the active
persona package through `PersonaRegistry`, places the selected `persona.md`
first, then appends task guidance and the **User-facing voice contract**.  The contract tells the model to:

- keep the active persona for ordinary replies and failures;
- start user-visible errors with a clear `⚠️ **Error — ...**` marker;
- preserve operation names, tool names, and technical details;
- use only light nautical humor so diagnostics stay readable.

Planning, ranking, routing, summarization, and reflection prompts are not
normally user-facing.  They may stay sterile and machine-oriented when they are
expected to return JSON, labels, or scoring data.  Do not add persona flavor to
strict machine prompts unless their output is directly shown to the user.

## Deterministic error path

Some failures bypass the LLM.  These must use `AgentErrorPresenter` instead of
raw strings such as `Error: {e}` or `Assistant Error: ...`.

Use it for:

- agent loop exceptions;
- streaming loop exceptions;
- plan-step failures;
- queue/download failures;
- circuit-breaker messages;
- websocket/bridge exceptions.

The presenter keeps a consistent shape:

```markdown
⚠️ **Error — short title**
Captain, I hit a snag. No need to shoot the parrot yet.
**Operation:** subsystem or tool
**Details:** `exact technical detail`
**Next step:** optional recovery advice
```

## Extension rule

When adding a new user-facing path, ask:

1. Is this text generated by the LLM for the user? Use `PromptBuilder`.
2. Is this a deterministic error shown to the user? Use `AgentErrorPresenter`.
3. Is this a machine-only prompt/result? Keep it precise and persona-neutral.

Round 27 adds `scripts/reliability_audit_round27.py` to enforce the main
contract and prevent the most common regression: raw sterile chat errors.
